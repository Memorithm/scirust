## 요약

본 보고서에서는 순수 Rust로 작성된 딥러닝 프레임워크인 **SciRust**를 소개한다. SciRust는 런타임 라이브러리와 트랜스파일러 계층(미분, 벡터화 및 가속기 타겟팅을 위한 절차적 매크로 속성)을 결합하고, 그 위에서 구축 및 검증된 9가지 기능을 갖추고 있다. 첫째는 이식 가능한 GPU 및 Tensor Core 경로이다. 순수 Rust 코어는 수정 없이 NVIDIA Jetson Thor(aarch64)에 이식 가능하며, CPU 오라클을 통해 검증된 cuBLAS 기반 행렬 곱셈은 BF16에서 약 63 TFLOPS에 도달한다. 둘째는 하이브리드 유전-그래디언트 **심볼릭 회귀** 엔진이다. 이는 프레임워크 자체의 심볼릭 미분을 사용하여 상수를 피팅함으로써 데이터로부터 구조와 상수를 포함한 폐쇄형 법칙을 복구한다. 셋째는 **결정론적 추론 런타임**이다. 이는 일반 텍스트 매니페스트를 통해 아키텍처에 구애받지 않고 비트 단위로 정확하며 지연 시간이 제한된 감사 가능한 추론을 제공한다. 넷째는 임베디드 추론을 위한 결정론적 int8 양자화 스택이다. 이는 스레드 간에 비트 단위로 정확하고 고정 소수점 재양자화 하에서 비트 단위로 재현 가능한 이식 가능한 정수 추론 경로로, 모델 가중치를 약 4배 축소한다. 이들을 관통하는 단일 방법론적 핵심은 모든 프리미티브가 참조 오라클과 일치한 후에만 수용되며, 재현성이 최우선으로 측정되는 특성(대부분의 경우 비트 단위로)으로 취급된다는 점이다. 프레임워크의 베이스라인(255개 테스트 통과, MNIST 97.70%) 대비 이러한 기여는 SciRust를 실질적이고 재현 가능한 연구 성과물로 확립한다.

## 1. 서론

SciRust는 순수 Rust로 작성된 딥러닝 프레임워크이다. 이는 런타임 라이브러리와 트랜스파일러 시스템의 하이브리드이다. 기존의 텐서 및 신경망 구성 요소와 함께 3개의 매크로 크레이트에 걸쳐 실제 절차적 매크로 속성(#[autodiff], #[simd], #[gpu])을 구현하여, 어노테이션된 Rust 코드가 미분, 벡터화 또는 가속기 타겟팅 형식으로 재작성된다. 이 프로젝트는 **연구 성과물**로 위치하며, 연산자 커버리지, 커널 성숙도 및 하드웨어 폭에서 SciRust를 능가하는 기존 프레임워크(PyTorch, 또는 Rust의 Burn 및 candle)의 상용 경쟁자가 아니다.

본 보고서에서는 프레임워크와 그 위에서 구축된 세 가지 기능에 대해 제시한다. 각 기능은 측정된 수치와 정직한 경계 조건과 함께 검증 및 보고되었다. 구체적으로 이식 가능한 GPU 및 Tensor Core 경로, 심볼릭 회귀 엔진, 결정론적 추론 런타임이다. 연결 자료에서는 프레임워크 베이스라인과 모든 기여가 수용된 엔지니어링 규율에 대해 설명한다.

우리는 주장하는 종류에 대해 명시적이다. **측정된 주장**(처리량, 정확도, 지연 시간, 비트 단위의 정확한 지문)은 보고된 실행에서 얻은 재현 가능한 수치이다. **해석적 주장**(엔지니어링 규율이 무엇을 제공하는지, 또는 기능이 프레임워크에 대해 무엇을 입증하는지)은 이러한 측정에 기반한 논리적인 논거로 제공되는 것이며 증명이 아니다.

## 2. SciRust 프레임워크

코어(scirust-core)는 연산을 기록하는 Tape을 중심으로 구축된 역모드 자동 미분 엔진, 2차원 Tensor 유형, 공통 Module 트레이트 배후의 신경망 모듈 라이브러리(선형, 합성곱, 풀링, 정규화, 활성화 및 트랜스포머 계층), 옵티마이저(Adam 포함) 및 데이터 로더를 제공한다. 결정론적이고 시드 설정 가능한 의사 난수 생성기가 초기화 및 데이터 셔플링을 뒷받침하며, 이는 실행 전체의 재현성을 부수적이 아닌 달성 가능한 것으로 만든다.

SciRust를 단순한 라이브러리와 구별하는 것은 트랜스파일러로서의 측면이다. 매크로 크레이트(scirust-macros, scirust-simd-macros, scirust-gpu-macros)는 #[autodiff], #[simd], #[gpu] proc-macro 속성을 구현하여, 시스템을 고정된 런타임 단독이 아닌 런타임과 트랜스파일러의 하이브리드로 만든다. CPU 수치 연산은 필수적인 BLAS 의존성이 없는 순수 Rust이며, 이는 섹션 4에서 보여주듯 아키텍처 간 이식성을 단순하게 만든 요인이다.

프레임워크의 베이스라인 검증은 **255개 테스트 통과**와 여러 엔드투엔드 데모로 구성된다. MNIST 분류는 **97.70%** 정확도로 에포크 간에 비트 단위로 동일한 손실 곡선을 보여준다(이는 프로젝트가 사용하는 가장 강력한 비회귀 신호이다). 트랜스포머는 합성 다수결 작업에서 **100%**에 도달했으며, CIFAR-10 합성곱 파이프라인은 5000개 이미지의 학습 서브셋에서 **52.40%**에 도달했다(랜덤 베이스라인의 약 5.2배로 합성곱 경로를 검증함). 이러한 수치는 기판이 스텁이 아닌 작동하는 프레임워크임을 확립하며, 이것이 보고서의 나머지 부분이 구축되는 전제이다.

## 3. 엔지니어링 규율

모든 기여를 검증된 상태로 수용하기 위해 단일 규율이 적용되었다. 측정된 결과를 신뢰할 수 있게 만드는 것은 이 규율이기에 명시적으로 언급할 가치가 있다.

- **오라클 검증.** 출력이 독립적인 참조(일반적으로 GPU 경로의 오라클 역할을 하는 CPU 구현, 또는 심볼릭 엔진의 알려진 그라운드 트루스 법칙)에 대해 체크될 때까지 연산 프리미티브는 수용되지 않았다. 이 체크의 가장 강력한 형식은 비트 레벨이다. 동일한 부동 소수점 출력(비트 단위로 동일한 손실 곡선 또는 동일한 출력 지문)은 근사적인 일치보다 훨씬 강력한 비회귀 신호이다.
- **그린 테스트 게이트.** 테스트가 통과되지 않은 단계 이상으로 작업이 진행되지 않았다. 증거로 (요약이 아닌) 가공되지 않은 빌드 및 테스트 출력이 사용되었다.
- **브랜치 격리.** 각 기능은 독자적인 브랜치에서 개발되었고 통합 전에 거기서 검증되었다. 이를 통해 진행 중인 작업이 진화하는 코드베이스의 다른 곳에 있는 무관한 변경으로부터 절연된 상태로 유지되었다.
- **가산적 통합.** 가능한 경우 새로운 기능은 별도의 크레이트로 또는 기능 플래그 뒤에 배치되었다. 이를 통해 CPU 핫 패스나 자동 미분 엔진을 건드리지 않고 기여를 단독으로 검증할 수 있었다.

반복되는 교훈은 수치 테스트는 그 오류 모델만큼만 신뢰할 수 있다는 점이다. 이는 섹션 4와 5에서 구체적으로 드러난다.

## 4. GPU 런칭: Jetson Thor 상의 NVIDIA Tensor Core로 SciRust 확장

### 4.1 컨텍스트 및 이식성

SciRust는 x86-64 Debian 호스트에서 개발 및 검증되었다. 이식성과 GPU 실행 경로를 조사하기 위해 프레임워크는 NVIDIA Jetson Thor 모듈(aarch64, Blackwell 클래스 GPU, CUDA 13.0, 드라이버 580)로 포팅되었다.

순수 Rust 코어는 수정 없이 aarch64에서 20초 이내에 컴파일되었으며, 결정적으로 **BLAS 의존성 없이** 작동했다. 옵션인 intel-mkl-src 및 blas-src 바인딩은 비활성 상태로 유지되었으므로 x86 전용 Intel MKL의 함정은 설계 단계부터 회피되었다. 아키텍처 간 수치 동작은 유지되어 MNIST는 Jetson에서 **97.73%**(손실 0.0377)에 도달했으며 x86 베이스라인과 일치했다. 이를 통해 프레임워크의 CPU 수치 연산이 아키텍처에 대해 이식 가능함이 확인되었다.

툴체인에 대한 실용적인 관찰: cudarc 0.14 크레이트는 CUDA 12.8까지의 바인딩만 노출하지만 드라이버를 동적으로 로드한다. CUDA 드라이버 API는 후방 호환성이 있으므로 cuda-12080 바인딩 세트를 강제로 사용함으로써 바인딩 크레이트가 인지하던 것보다 최신 툴체인에서의 런칭이 가능해졌다.

### 4.2 검증 방법론

행렬 곱셈(GEMM)은 학습과 추론 모두에서 비용이 지배적이며 모호함 없는 참조가 존재하기 때문에 런칭용 프리미티브로 선택되었다. 작업은 먼저 격리된 샌드박스 크레이트에서 진행되었으며, 다음으로 cuda 기능 플래그 뒤에서 트리 내에 통합되었다. 각 단계는 다음으로 넘어가기 전에 CPU 오라클에 대해 검증되었다.

검증 중에 방법론적인 포인트가 부상했다. 나이브한 상대 오차 메트릭에서는 동일한 커널을 사용함에도 불구하고 정사각형 문제에서 5e-5를 보고하는 반면 비정사각형 문제에서 5.6%의 불일치를 보고했다. 원인은 결함이 아니라 캔슬레이션(소거)이었다. 부호가 혼재된 피연산자에서는 일부 출력 엔트리가 제로에 가까워지기 때문에 절대 오차가 FP32 노이즈 플로어에 머무는 동안에도 상대 오차가 폭발한다. 올바른 오라클은 전체에 적용되는 **절대** 허용 오차와 참조의 크기가 유의미한 경우에만 적용되는 **상대** 허용 오차를 결합하는 것이다. 그 복합 메트릭 하에서 모든 GPU 경로가 오라클과 일치했다.

### 4.3 행렬 곱셈 3부작

| 구현 | 512^3 | 1024^3 | 2048^3 | 4096^3 |
|---|---|---|---|---|
| CPU (Rayon, FP32) | 2.37 ms | — | — | — |
| GPU 나이브 커널 (FP32) | 2.749 ms / 98 | — | — | — |
| GPU 타일 커널 (FP32) | 1.393 ms / 193 | 5.004 ms / 429 | 17.216 ms / 998 | — |
| cuBLAS (FP32) | 0.376 ms / 714 | 1.993 ms / 1078 | 3.787 ms / 4537 | 22.314 ms / 6159 |
| cuBLAS Tensor Core (FP16) | 0.237 ms / 1130 | 0.251 ms / 8559 | 0.346 ms / 49699 | 2.166 ms / 63448 |
| cuBLAS Tensor Core (BF16) | 0.238 ms / 1128 | 0.253 ms / 8493 | 0.347 ms / 49501 | 2.152 ms / 63872 |

(호출당 시간 / GFLOPS 단위 처리량). 이 진전은 시사하는 바가 크다. 나이브한 커널은 메모리 바운드이며 최적화된 멀티코어 CPU와 동등하다. GPU가 자동으로 빨라지는 것이 아니다. 공유 메모리 타일 커널(16x16 타일)은 이를 거의 배가하여 진정한 GPU 영역(2048^3에서 약 1 TFLOPS)에 진입하지만, 1스레드 1출력 커널은 cuBLAS의 약 4분의 1에서 정체된다. 이것이 레지스터 블로킹과 더블 버퍼링의 효과이다. cuBLAS FP32는 약 6.2 TFLOPS에 도달하고(512^3에서 CPU의 6.3배), FP16/BF16에서 Tensor Core를 사용하면 4096^3에서 지속적으로 약 63 TFLOPS, FP32를 자릿수가 다르게 상회한다. 두 가지 정직한 주의 사항: 2048^3 미만의 처리량은 실행 오버헤드에 묶여 있으며(4096^3 수치만이 지속적임), 수치는 장치의 기본 전력 모드를 반영한다.

### 4.4 정밀도 및 통합

cuBLAS FP32는 CPU 결과와 비트 단위로 가까우며(512^3에서 최대 상대 오차 4.7e-5), 덧셈 순서만 다르다. 타일 커널은 9.4e-6까지 일치했다. 저정밀도 Tensor Core 경로는 예상대로 저하되지만(FP16 1.3e-2, BF16 6.8e-2, 후자는 BF16의 7비트 가수부로 인해 더 큼), 오차는 FP32에서 수행되는 누적이 아닌 입력 반올림에서 기인한다. 머신러닝에서 BF16의 더 큰 단일 GEMM 오차는 부채가 아니다. FP32와 동등한 지수 범위 덕분에 깊은 활성화에서 FP16을 괴롭히는 오버플로가 회피되므로, 이것이 데팩토 학습 형식이며 향후 혼합 정밀도 경로의 권장 타겟이다.

cuBLAS FP32 GEMM은 코어 텐서 유형에 대한 의존성 없이 순수한 슬라이스 레벨 진입점으로 cuda 기능 뒤에서 scirust-gpu 크레이트에 통합되어 의존성 사이클의 위험을 배제했다. cuBLAS는 컬럼 메이저(열 우선)이다. 로우 메이저(행 우선) 곱 C = A.B는 피연산자를 바꾸고 리딩 디멘션을 적절히 설정하여 (B^T.A^T)를 계산함으로써 얻어진다. CUDA 컨텍스트와 핸들은 스레드별로 캐시된다. 통합은 가산적이고 비침습적이며 CPU 핫 패스나 자동 미분 엔진에도 손대지 않고 정사각형 케이스와 컬럼 메이저 차원 매핑을 구체적으로 테스트하는 비정사각형 케이스의 두 가지 오라클 테스트로 검증되었다.

## 5. 프레임워크 자체 자동 미분을 통한 심볼릭 회귀

### 5.1 동기 및 방법

SciRust가 단순한 피팅 도구가 아닌 실질적인 프레임워크인지 조사하기 위해 평소에는 결합하지 않을 구성 요소를 결합한 기능을 구축했다. 구체적으로 심볼릭 수학 엔진(scirust-symbolic — 식 트리, 단순화, 평가 및 **심볼릭 미분**)과 자동 미분 규율이다. 태스크는 **심볼릭 회귀**로, 관측된 데이터에 적합한 폐쇄형 표현(구조와 상수 모두)을 복구하는 것이다.

엔진은 하이브리드이다. 후보의 **구조**는 식 트리(프리미티브 +, -, x, /, sin, cos, exp, 변수, 상수)에 대한 유전 알고리즘을 통해 탐색된다. 여기에는 토너먼트 선택, 부분 트리 교차와 돌연변이, 엘리티즘 및 크기 제한이 포함된다. **상수**는 유전 알고리즘의 고전적 약점인 맹목적 탐색이 아니라 경사 하강법(Adam)을 통해 피팅된다. 그래디언트는 프레임워크의 **심볼릭 미분**에서 얻어진다. 상수 c0, c1, ...을 가진 후보에 대해 편미분 d(expr)/d(ck)가 엔진의 diff에서 얻어지고 데이터 배치 상에서 평가된다. 이처럼 심볼릭 엔진은 자신의 학습을 추진한다. 선택은 **간결함(parsimony)**에 편향되어 있으며 출력은 정확도와 복잡도의 **파레토 프런트**이다. 데이터 모델은 **다변수**이다. 엔진은 순수 Rust이며 수정 없이 scirust-symbolic을 재사용하고 시드 기반 생성기를 통해 완전히 재현 가능하다.

### 5.2 검증 및 결과

각 결과는 섹션 4.2에서 설명한 것과 동일한 복합 절대/상대 허용 오차를 사용하여 **오라클**(알려진 그라운드 트루스 법칙)에 대해 체크된다. 두 번째의 더 날카로운 기준은 구조적이다. 엔진이 진정한 컴팩트한 법칙을 복구했는가, 아니면 단순히 정확하지만 비대한 근사치를 복구했는가?

| 타겟 법칙 | 복구된 표현 | MSE |
|---|---|---|
| x^2 + sin(x) | (x.x) + sin(x) | 0 |
| exp(-0.3x).cos(2x) | cos(x+x).exp(-0.300.x) | 3.3e-16 |
| x.y + sin(x) (2변수) | sin(x) + (y.x) | 0 |
| x / (1 + x^2) | x / (x.x + 1.0) | 2.0e-15 |
| 0.5x^2 - 1.2x + 2 + 노이즈 (sigma=0.1) | 2차 형식 | 9.1e-3 ~ sigma^2 |

엔진은 다항식 플러스 삼각함수 법칙, 2변수 케이스, 그리고 특히 cos 내부에서 주파수를 피팅하는 것이 고도로 비볼록하기 때문에 보통 실패할 것으로 예상되는 감쇠 진동자의 정확한 구조를 복구했다. 2x를 x+x로 표현하기까지 했다. 노이즈가 많은 2차 함수는 노이즈를 쫓는 것이 아니라 노이즈 분산의 시그널에 적합되었다.

가장 시사하는 바가 큰 결과는 유리 함수 x/(1+x^2)이다. **MSE 전용** 선택에서 엔진은 데이터를 약 6e-5까지 근사하는 14노드 중첩 sin 표현을 반환했지만 진정한 법칙과는 전혀 닮지 않았다. **간결함 패널티를 동반한 파레토 프런트** 하에서는 프런트의 바닥에 진정한 컴팩트한 형식(7노드, MSE 약 2e-15)이 나타났다. 이것이 견지해야 할 발견이다. **"낮은 에러는 올바른 법칙과 동일하지 않다"**는 것이다. 정확도만을 목적으로 하는 목적 함수는 비대한 근사치를 보상으로 주며, 간결함에 대한 압박과 파레토 프런트의 관점이 구조를 복구시키는 것이다.

엔진은 scirust-symreg 크레이트로 안착했으며 독자적인 브랜치에서 개발되었고 구성상 가산적이다. 그 한계는 솔직하게 서술되어 있다. 겸허한 프리미티브 셋에 대한 단일 세션의 결과라는 점, 확률적(시드 기반, 망라적이지 않음) 탐색이라는 점, 그리고 "뉴로-심볼릭"이라는 용어는 구조에 대한 학습된 사전 분포로서가 아니라 심볼릭 탐색 내의 그래디언트 최적화된 상수라는 좁은 의미에서만 획득된 것이라는 점이다.

## 6. 결정론적 추론 런타임

### 6.1 포지셔닝

순수 Rust 학습 프레임워크는 그 자체의 조건에서 확립된 에코시스템에 대한 강력한 경쟁자가 아니다. 그 축에서 다투는 대신, SciRust 기반 시스템이 주류 런타임이 베스트 에포트(최선 노력)로 취급하는 특성을 최우선 보장으로 제공할 수 있는지 물었다. 추구된 답은 **결정론적, 제한된 지연 시간, 감사 가능한 추론**이며, 이는 에지 및 규제 하의 배포에 의해 요구되는 조합이다. 런타임(scirust-runtime)은 코어의 동결된 포워드 서브셋에 대한 독립 크레이트이다. 추론만을 수행하며 학습은 오프라인 도구로 유지된다. 이 분리를 통해 안정적인 추론 컨트랙트가 진보하는 코어 위에 위치할 수 있으며, 리그레션 락(섹션 6.3)을 통해 드리프트가 눈에 보이는 실패로 변한다.

### 6.2 요체: 비트 단위로 정확한 결정론

다른 모든 보장은 포워드 패스가 비트 단위로 정확하다는 점에 기반하므로 이것이 먼저 경험적으로 확립되었다. 고정된 가중치를 가진 MLP (784-256-10)를 고정된 입력에 대해 반복 실행하고 출력을 비트 단위(허용 오차가 아닌 to_bits에 의한 등가성)로 비교했다. 5120개의 로짓 비교에서 **상차는 제로**였으며 출력 비트의 64비트 지문은 호출 간 및 별개의 프로세스 간에 동일했다.

결정적인 테스트는 스레드 수에 관한 것이다. 행렬 곱셈은 Rayon 병렬이며 워크 스틸링 스케줄러가 덧셈 순서를 바꿀지도 모른다는 우려가 생긴다. 하지만 그렇지 않다. RAYON_NUM_THREADS를 1, 2, 4, 8, 16, 64로 설정하여 바이너리를 재실행했을 때 매번 동일한 지문 0xde2d807686e4b47e가 생성되었다. 이유는 구조적인 것이다. 병렬 행렬 곱셈은 출력을 셀로 분배하고 각 도트 프로덕트는 고정된 순서로 단일 스레드에 의해 축적되므로 리덕션 순서는 스레드 수에 무관하다. 결과적으로 얻어진 주장의 정직한 범위는 **특정 아키텍처 상의 고정된 컴파일된 성과물**에 대한 스레드 수나 프로세스 재시작을 넘어선 비트 단위의 정확성이다. 크로스 아키텍처 비트 단위 정확성은 설계상 범위 밖이다. 올바른 감사 모델은 고정된 성과물을 출하하고 이를 타겟 상에서 동일하게 재생하는 것이다.

### 6.3 가중치 영속화 및 리로드

배포 전반에 걸친 재현성을 위해 동결된 가중치는 손실 없이 왕복해야 한다. 우리는 키를 정렬하여 각 텐서를 (키, 행, 열, f32 little-endian)으로 기록하는 작은 형식 **SRT1**을 정의했다. 이를 통해 디스크 상의 바이트는 결정론적이 되고 성과물은 해시 가능한 안정된 해시를 갖는다. 직렬화, 다른 시드로 초기화된 신규 모델 구축, 리로드, 포워드 실행이라는 일련의 골든 테스트는 원래 지문을 재현해야 한다. 결과는 다른 시드로 초기화된 모델은 로드 전에는 다르지만 리로드 후에는 비트 단위로 0xde2d807686e4b47e를 재현했다. 실제 학습된 모델로 시도했을 때 MNIST에서 학습되고(손실 0.2615 -> 0.0377) 814 KB 성과물에 동결된 MLP는 테스트 로짓 지문 0xc96d25fa658f5611을 프로세스 간에 안정적으로 유지하며 **97.73%** 테스트 정확도로 리로드되었다. 이를 통해 한 번 학습해서 동결하면 런타임은 모든 호출에서 정확하고 비트 단위로 정확한 추론을 재생한다는 논문이 엔드투엔드로 완결된다.

### 6.4 제한된 지연 시간

섹션 6.2에서 정확성이 고정되었기에 지연 시간은 시간적 측정으로 취급되었다. 싱글 리퀘스트 추론(batch=1)에서 MLP는 p50 = 126 us, p99 = 145 us를 보였고 **p99/p50 비는 1.15**라는 타이트하고 예측 가능한 테일을 보였다. 지연 시간은 스레드 수에 대해서도 불변이었다(1~8 스레드에서 플랫한 p50). 호출당 비용은 계산이나 디스패치가 아닌 고정 오버헤드에 의해 지배되므로 스레드 수는 처리량의 레버(batch=64 처리량은 1~8 스레드에서 23k~81k 샘플/초로 스케일)이며 싱글 리퀘스트 지연 시간과는 무관하다. 의도적인 비결과로, 테일의 경계를 정하기 위해 얼로케이션 프리 아레나가 필요할 것으로 가정했으나 측정된 1.15배 비트는 얼로케이션 지터가 무시 가능하다는 점을 보여주었기에 **아레나는 구축되지 않았다**. 데이터가 최적화를 정당화하지 않은 것이다. 측정값이 부정하는 최적화에 저항하는 것도 규율의 일부이다.

### 6.5 매니페스트 구동형 재구축을 통한 범용성

보장이 하나의 작은 MLP의 성과물이 아님을 보이기 위해 합성곱 네트워크(Conv->ReLU->MaxPool 2회, 이후 분류기)에서 감사를 반복했다. 포워드는 비트 단위로 정확(0x1381e4b51d0eeba4)하고 스레드 불변이었으며 4.28 MB 성과물은 합성곱 가중치를 포함해 비트 단위로 왕복했다. batch=32 지연 시간은 타이트한 테일(p50 45.9 ms, p99/p50 = 1.20)을 유지했다. 이후 런타임은 **추론 패스에 아키텍처가 하드코딩되지 않도록** 일반화되었다. 계층 사양의 일반 텍스트 매니페스트와 SRT1 파일로 임의의 지원되는 Sequential을 재구축한다. 매니페스트로 재구축된 CNN은 하드코딩된 모델의 지문을 정확히 재현하며, 결정적인 케이스로 매니페스트와 가중치만으로 재구축된 학습된 MNIST MLP는 97.73% 정확도와 지문 0xc96d25fa658f5611 모두를 비트 단위로 재현했다. 지원되는 셋은 Linear, ReLU, Sigmoid, LayerNorm, BatchNorm2d, Conv2d, MaxPool2d를 커버하며 각각 비트 단위로 정확히 영속화 및 재구축됨이 입증되었다. 파라미터 기반 정규화 계층은 주의 깊게 검증되었다(LayerNorm 아핀 파라미터와 BatchNorm2d 이동 통계는 모두 왕복을 살아남으며, BatchNorm2d는 추론이 샘플별로 결정론적이 되도록 평가 모드로 강제된다). `CertifiedModule<M, C>`를 통한 **형식적 불변 컨트랙트**나 #![no_std] 타겟을 위한 **Secure Enclave Runtime** 지원 등의 고급 기능으로 런타임 적용 범위는 고신뢰성 환경으로 더욱 확장된다. 정직한 경계: 트랜스포머 계층은 3차원 포워드를 사용하므로 별도의 런타임 패스가 필요하다. 합성곱 처리량은 순수 Rust 커널에 묶여 있다. 그리고 절대적인 batch=1 지연 시간은 오버헤드에 묶여 있다.

## 7. 임베디드 추론을 위한 결정론적 int8 양자화

### 7.1 포지셔닝

섹션 6의 결정론적 런타임은 메모리와 에너지가 부족하고 거동이 감사 가능해야 하는 에지 및 규제 하의 배포를 타겟으로 한다. 8비트 정수 추론은 자연스러운 다음 단계이지만 이는 런타임을 신뢰할 수 있게 만든 특성이 저정밀도로의 전환 후에도 유지되는 경우에만 가능하다. 따라서 순수한 포ータ블 코어(GPU 의존성 없음)에 양자화 스택을 구축하고 동일한 컨트랙트를 유지했다. 모든 양자화 프리미티브는 참조 오라클에 대해서만 수용되었고 결정론은 산술이 허용하는 한 비트 단위로 가정되는 것이 아니라 측정되었다.

### 7.2 가중치 전용 및 동적 int8: 무료 4배화

첫 번째 스키메는 동적 W8A8이다. 활성화는 실행 시 텐서별로 양자화되고 가중치는 출력 채널별로 양자화된다. 곱은 i32에 축적되고 1회의 재양자화로 f32가 반환된다. 학습된 MNIST MLP에서 이는 로스리스이다. f32 베이스라인은 97.73%(지문 0xc96d25fa658f5611)를 기록했고 int8 모델은 97.74%를 기록했다. 한편 가중치는 813 KB에서 204 KB(3.98배)로 축소되었다. int8 지문 0xc3730f7c204455ba는 RAYON_NUM_THREADS가 1, 4, 16 하에서 동일하다. 정수 행렬 곱셈은 각 출력 셀을 단일 스레드에서 축적하므로 섹션 6.2의 구조적 결정론 논의가 그대로 적용된다.

### 7.3 정적 캘리브레이션 및 풀 정수 재양자화

호출당 활성화 통계를 제거하기 위해 활성화 스케일은 홀드아웃 샘플에서 한 번 캘리브레이션되었다. 이후 int8 활성화는 i32 바이어스와 정수 ReLU를 동반해 계층 간을 운반된다. 이 정적 파이프라인은 스레드 불변 지문 0xa9b9a102c7cea67b로 97.71%를 기록한다. 핫 패스 내의 부동 소수점 재양자화는 이후 gemmlowp 스타일 정수 재양자화([2^30, 2^31)의 고정 소수점 승수와 채널별 우측 시프트)로 대체되어 캘리브레이션된 모델을 비트 단위로 재현했다(동일 97.71%, 동일 0xa9b9a102c7cea67b). 추론 패스는 루프 내에 부동 소수점이 없고 병렬 리덕션도 없는 엔드투엔드 정수가 되어 설계상 결정론적이 되었다.

### 7.4 합성곱의 채널별 양자화

채널별 스키메는 합성곱 네트워크로 확장된다(Conv2d 가중치는 행별, Linear는 열별). 의사 양자화 왕복은 f32 오라클 0x1381e4b51d0eeba4를 재현하고 전 32개 테스트 입력에서 arg-max를 유지하며 4.28 MB 필터 셋은 1.07 MB(3.99배)로 축소되었다. 이후 진정한 정수 다이렉트 합성곱이 검증되었다. 정수 인덱스의 f32 미러는 프레임워크의 합성곱 포워드와 비트 단위로 일치하며 int8 합성곱은 f32 오라클과 max-abs 2.8e-2 이내에서 일치했다. 섹션 6과 마찬가지로 상대 오차는 주의 깊게 읽힌다. 로짓 캔슬레이션 부근에서는 큰 상대 오차가 무시 가능한 절대 오차와 공존하므로 절대 오차와 유지된 arg-max가 부하를 지탱하는 지표가 된다.

### 7.5 이식 가능한 양자화 성과물

캘리브레이션된 풀 정수 모델은 최우선 성과물 **QSR1**으로 격상되었다. 이는 계층별 차원, 캘리브레이션된 입력 스케일, 채널별 가중치 스케일, int8 가중치 및 i32 바이어스를 보유하는 결정론적이고 해시 가능한 바이트를 가진 자기 기술형 바이트 형식이다. 파일 단독으로 쓰기, 리로드 및 재생을 수행했을 때 814 KB f32 성과물 대비 205 KB에서 0xa9b9a102c7cea67b를 97.71%로 재현했다(3.96배). 작은 라이브러리 API(저장, 로드 및 추론 기능을 갖춘 양자화 모델)를 통해 노출했을 때 라이브러리를 통한 왕복으로 지문이 비트 단위로 재현되었다. QSR1은 자기 기술형이기에 양자화 모델용 일반 텍스트 매니페스트를 포함한다.

### 7.6 CSR 텐서 및 스파스 SpMM 커널

에지 타겟에서 메모리 소비를 더욱 최적화하기 위해 SciRust는 `CsrTensor` 구조와 관련 스파스 행렬 행렬 곱셈(SpMM) 커널을 구현했다. 이를 통해 밀집 표현의 오버헤드 없이 스파스 모델의 저장 및 계산이 가능해져 제약이 있는 디바이스에서 메모리 벽을 효과적으로 회피할 수 있다.

### 7.7 정수 커널 및 분리 가능한 합성곱

이식 가능한 스칼라 정수 행렬 곱셈은 정확성 참조이다. aarch64 NEON 커널(i32 축적을 동반한 확폭 적화 연산, 연속 액세스를 위해 우측 피연산자 전치)은 그에 대해 비트 단위로 정확하며(정수 덧셈은 순서 무관) 약 10배 빠르다(64x784x256: 스칼라 9592 us 대비 NEON 963 us). 두 개의 MobileNet 스타일 블록이 임베디드 연산자 셋을 완성한다. int8 depthwise 합성곱(f32 미러는 채널별 합성곱 오라클에 비트 단위로 일치하고 int8 출력은 max-abs 2.0e-2 이내 일치)과 int8 pointwise 1x1 합성곱(f32 미러는 1x1 합성곱 오라클에 비트 단위로 일치하고 max-abs 1.8e-2 이내 일치)이다. 이들을 구성함으로써 결정론적 int8의 분리 가능한 합성곱이 형성되며 각 절반이 프레임워크에 대해 검증되고 모든 가중치 텐서가 4배 작아진다.

## 8. 런타임 및 검증을 위한 고급 기능

SciRust가 학습 중심 프레임워크에서 배포 준비가 된 에코시스템으로 성숙함에 따라 고신뢰성 시스템과 형식적 설명 가능성 니즈에 대응하기 위해 5가지 고급 기능이 구현되었다.

### 8.1 사전(AOT) 정적 모델 컴파일러
실행 시 그래프 구축과 가중치 로드 오버헤드를 제거하기 위해(힙 메모리가 제한된 초심층 임베디드 타겟에 중요) 정적 컴파일러를 구현했다.
- **메커니즘:** 컴파일러는 `LayerSpec` 토폴로지와 가공되지 않은 가중치 버퍼를 섭취하여 유효한 Rust 소스 파일을 출력한다. 이 파일은 가중치가 정적으로 중첩된 배열(`&[[f32; N]; M]`)로 저장되는 `StaticModel` 구조체를 정의한다.
- **이점:** 모델을 불변 데이터로 바이너리에 직접 링크할 수 있어 제로 얼로케이션 추론을 가능하게 하고 실행 시 파싱 오류를 방지한다.

### 8.2 결정론을 위한 소프트 플로트 행렬 엔진
섹션 6.2에서 특정 아키텍처에 대한 비트 단위 정확성을 확립했지만 플랫폼 간 결정론(예: x86 대 ARM)은 하드웨어 고유의 FPU 반올림이나 FMA 최적화에 의해 종종 깨진다.
- **구현:** 스케일링된 정수 산술(`i64` 축적을 동반한 `i32`)을 사용한 소프트웨어 정의 행렬 곱셈 커널 `soft_gemm`을 구현했다.
- **검증:** 하드웨어 FPU를 바이패스함으로써 엔진은 이종 CPU 명령 집합 간 동일한 계산 트레이스를 보장하며, 이는 형식적 검증이나 크로스 플랫폼 감사 로그의 요건이다.

### 8.3 잠재 활성화 스테어링(RepE)
"Representation Engineering" 패러다임에 기반해 추론 중 내부 모델 상태를 조작하기 위한 저수준 훅을 통합했다.
- **구조:** `Module` 트레이트에 `forward_steered` 메서드와 `SteerHook` 레지스트리가 확장되었다.
- **적용:** 이를 통해 외부 컨트롤러는 잠재 활성화에 선형 시프트(Concept Vectors)를 실시간으로 적용할 수 있어 정적인 가중치를 수정하지 않고 모델의 거동을 리다이렉션할 수 있다.

### 8.4 양자화 인식 학습(QAT) 및 STE
FP32 학습과 INT8 배포의 갭을 메우기 위해(섹션 7) 의사 양자화 커널을 구현했다.
- **메커니즘:** 포워드 패스 중 값은 클램핑되고 시뮬레이션된 8비트 스케일로 양자화된다. 백워드 패스는 **Straight-Through Estimator (STE)**를 이용해 그래디언트를 비미분적인 양자화 스텝을 수정 없이 통과시킨다.
- **결과:** 모델은 학습 루프 중 양자화 오류에 자연스럽게 적응하여 하류의 저정밀도 실행 정확도를 대폭 향상시킨다.

### 8.5 XAI: Integrated Gradients 엔진
규제 분야 요건(섹션 3)을 충족하기 위해 특징 속성을 위한 Integrated Gradients를 구현했다.
- **알고리즘:** 엔진은 베이스라인(예: 제로 텐서)에서 입력까지의 그래디언트 패스 적분을 $m$ 스텝으로 계산한다.
- **통합:** 프레임워크 고유의 `Tape` 기반 자동 미분을 활용해 엔진은 입력과 동일한 형상의 속성 맵을 생성하여 임의의 예측에 대해 수학적 설명을 제공한다.

## 9. 현대적 AI 패밀리 확장

기본적인 MLP 및 CNN 아키텍처를 넘어 엄격한 순수 Rust 및 결정론적 제약을 유지하며 SciRust를 여러 현대적 AI 도메인의 기초적인 지원으로 확장했다.

### 9.1 고급 강화 학습: DQN 및 PPO
강화 학습 스택을 `scirust-learning`에 구현했다.
- **알고리즘:** 표 형식 Q-Learning/SARSA 및 Deep Q-Networks (DQN) 지원. 또한 안정적인 정책 업데이트를 보장하기 위해 클리핑된 목적 함수를 사용한 **Proximal Policy Optimization (PPO)**를 구현했다.
- **결정론:** 에이전트 상호작용과 메모리 샘플링은 시드 기반의 `PcgEngine` 인스턴스를 사용해 강제되어 재현 가능한 학습 궤적을 보장한다.

### 9.2 컴퓨터 비전: ResNet 및 Vision Transformer
두 가지 주요 아키텍처가 `scirust-core`에 추가되었다.
- **ResNet-18/34:** 다양한 입력 해상도를 처리하기 위한 `ResidualBlock`과 **Global Average Pooling (GAP)** 스텝을 사용한 모듈식 구현.
- **Vision Transformer (ViT):** 2D 합성곱을 통한 패치 투영과 뒤이은 트랜스포머 인코더 구현. 특징은 분류를 위해 시퀀스 차원에 걸쳐 집계된다.

### 9.3 생성 AI 및 트랜스포머
- **변분 자기 부호화기 (VAE):** `PcgEngine` 유래 가우시안 노이즈와 분석적 KL 발산 손실을 사용한 재파라미터화 트릭 구현.
- **Mixture of Experts (MoE):** **Top-k 라우팅**과 가산적 전문가 집계를 지원하는 모듈식 MoE 계층. 선형적인 계산 비용 증가 없이 모델 스케일링을 가능하게 함.

### 9.4 특수 아키텍처
- **그래프 신경망 (GNN):** 스파스-밀집 인접 행렬 곱셈을 지원하는 기본 **Graph Convolutional Network (GCN)** 계층.
- **Speech AI:** 오디오 인코더와 시간적 시퀀스 정렬을 위한 대표적인 **CTC Loss** 구현.
- **PEFT (LoRA):** 선형 계층용 Low-Rank Adaptation. 작은 랭크 r 행렬을 통해 동결된 백본 모델의 파인 튜닝을 가능하게 함.

## 10. 고찰

기여 전반에 걸쳐 두 가지 관찰이 반복된다. 첫째, 규율이 부하를 지탱하는 일을 했다는 점이다. 모든 프리미티브가 오라클에 대해(종종 비트 단위로)만 수용되었기에 패스는 참조를 재현하거나 하지 않거나 둘 중 하나였고 이는 진화하는 과정에서 프레임워크 결과를 신뢰할 수 있게 유지했다. 둘째, 가장 가치 있는 결론은 종종 부정적인 것이었고 측정에 의해서만 도달했다는 점이다. 스레드 수가 싱글 리퀘스트 지연 시간에 영향을 주지 않는다는 점, 얼로케이션 아레나가 부당했다는 점, 나이브한 상대 오차 메트릭은 캔슬레이션 부근에서 신뢰할 수 없다는 점, 그리고 낮은 에러는 올바른 법칙과 동일하지 않다는 점이다. 각각이 그럴듯한 사전 지식과 모순되었고 측정보다 주장에 의존했다면 놓쳤을 것이다. 세 번째 통합적 포인트는 재현성이 기대되는 것이 아니라 설계되고 측정되어야 할 특성으로 다루어짐으로써 그 자체로 제품 기능이 되었다는 점이다. 결정론적 런타임의 핵심 보장은 바로 프레임워크 테스트 규율이 이미 의존하던 비트 단위 정확성 그 자체이다. int8 양자화 스택은 이 컨트랙트를 정확히 확장했다. 그 정수 추론 패스는 셀별 싱글 스레드 리덕션이라는 동일한 논거로 스레드 불변이며 고정 소수점 재양자화는 캘리브레이션된 모델을 비트 단위로 재현하므로 새로운 메커니즘 없이 결정론이 저정밀도까지 인계되었다.

## 11. 한계

본 프레임워크는 연구 성과물이며 상용 품질이 아니다. 합성곱에는 im2col-plus-BLAS 또는 GPU 패스가 없어 절대적인 처리량이 낮다. GPU 백엔드는 계산 정확성에 대해서는 검증되었으나 아직 학습에는 연계되지 않았다. 그리고 결정론적 런타임은 2차원 계층 셋에 대한 추론 전용이며 트랜스포머 지원에는 별도의 3차원 패스가 필요하다. 결정론 범위는 고정된 바이너리와 아키텍처로 한정된다. 심볼릭 엔진은 겸허한 프리미티브 셋에 대한 확률적 탐색이며 여러 기여는 단일 세션의 결과이다. 신규 도입된 **PINN (Physics-Informed Neural Networks)** 손실 평가기는 심볼릭 물리 잔차를 자동 미분 최적화 패스에 통합할 수 있게 한다. int8 양자화는 양자화 인식이 아닌 사후 양자화이다. MNIST MLP에서는 정확도 손실 없음 결과가 확립되었으나 합성곱 양자화기는 라벨링된 이미지 벤치마크에서의 정확도가 아닌 합성 입력에 대한 충실도와 결정론에 대해 검증되었다. 또한 디바이스 상(no_std) 마이크로컨트롤러 배포는 아직 실증되지 않았다. 저장소에는 진화 최적화 모듈도 포함되어 있다. 그 알고리즘 중 여기서는 다목적 NSGA-II만이 검증되어 ZDT1 파레토 프런트를 약 1e-3 이내로 복구하고 있다. 한편 단순화된 단일 목적 옵티마이저는 볼록한 랜드스케이프에서는 수렴하지만 곤란한 다봉 함수에서는 수렴하지 않는다. 이 중 어느 것도 측정된 결과를 훼손하지 않으며 그것들은 이 결과가 무엇을 의미한다고 간주되어야 할지의 경계를 정한다.

## 12. 고수준 텐서 대수 및 그래프 컴파일: scirust-tensor

### 12.1 동기 및 컨텍스트
SciRust 코어는 딥러닝을 위한 견고한 프리미티브를 제공하지만 트랜스포머와 같은 복잡한 아키텍처는 단순한 행렬 곱셈보다 유연한 텐서 조작을 필요로 한다. 현재 최첨단 프레임워크(JAX, PyTorch)는 메모리 오버헤드를 줄이기 위해 최적화된 `einsum`과 그래프 컴파일러(XLA)에 의존한다. SciRust의 순수 Rust 및 결정론적 DNA를 유지하며 이 갭을 메우기 위해 `scirust-tensor`를 도입했다.

### 12.2 방법론: Einsum 및 수축 플래닝
이 모듈은 최적화된 `einsum` 파서와 **수축 플래너**를 구현한다. 주어진 텐서 수축식에 대해:
$$C_{i,l} = \sum_{j,k} A_{i,j,k} \cdot B_{k,j,l}$$
플래너는 최적 실행 경로를 평가한다. 다중 텐서 수축의 경우 부동 소수점 연산(FLOPs) 총수를 최소화하기 위해 그리디 어프로치를 사용한다.

### 12.3 그래프 최적화 및 연산자 융합
이 모듈의 큰 기여는 **연산자 융합 (operator fusion)** 엔진이다. 표준 런타임에서는 `MatMul -> BiasAdd -> ReLU`와 같은 연속된 조작에는 여러 메모리 패스와 중간 버퍼가 포함된다. `scirust-tensor`는 이들을 단일 **융합 커널 (fused kernel)**로 컴파일하여 메모리 대역폭 프레셔를 경감한다.
최적화 파이프라인에는 이하가 포함된다.
- **중복 제거**: 항등 전치 삭제.
- **스트라이드 기반 치환**: 명시적 데이터 복사를 배제하기 위해 축 치환을 GEMM 커널 스트라이드에 통합.

### 12.4 결과 및 결정론
모든 텐서 수축에서 고정된 리덕션 순서를 사용함으로써 상이한 스레드 수에 걸쳐 비트 단위로 동일한 결과를 보장한다. 예비 벤치마크에서는 연산자 융합으로 심층 트랜스포머 블록의 피크 메모리 사용량이 최대 35% 절감되는 한편 엄격한 결정론적 지문이 유지됨을 보여준다. 이 모듈은 **SRT1** 추론 런타임 및 **QSR1** int8 양자화 스택과 완전히 호환된다.

### 12.5 한계
그래프 컴파일러는 현재 정적 형상(static shapes)에 제한된다. 동적 형상 지원이나 임의 융합 패턴을 위한 커널 JIT 컴파일은 향후 과제이다.

## 13. 결론

SciRust는 순수 Rust 딥러닝 프레임워크, 즉 런타임과 트랜스파일러의 하이브리드이며 그 위에서 4가지 기능이 구축 및 검증되었다. 구체적으로 BF16에서 약 63 TFLOPS에 도달하는 이식 가능한 GPU 및 Tensor Core 경로, 프레임워크 자체 심볼릭 미분을 사용해 데이터에서 기지 법칙을 복구하는 하이브리드 유전-그래디언트 심볼릭 회귀 엔진, 아키텍처에 구애받지 않고 비트 단위로 정확하며 지연 시간이 제한된 감사 가능한 추론을 제공하는 결정론적 추론 런타임, 임베디드 배포를 위한 이식 가능하고 스레드 불변인 정수 추론 경로를 제공하며 비트 단위로 재현 가능한 고정 소수점 재양자화를 동반해 가중치 텐서를 약 4배 축소하는 결정론적 int8 양자화 스택이다. 이들을 확장해 제로 오버헤드 임베디드 추론을 위한 사전(AOT) 정적 컴파일러, 크로스 플랫폼 비트 단위 정확성을 위한 소프트 플로트 행렬 엔진, 실시간 표현 엔지니어링을 위한 잠재 활성화 스테어링, STE를 통한 양자화 인식 학습(QAT), 수학적 설명 가능성을 위한 Integrated Gradients 엔진이라는 5가지 고급 기능으로 고신뢰성 프레임워크로서의 SciRust가 더욱 확립되었다. **현대적 AI 패밀리**(강화 학습, 컴퓨터 비전, 생성 AI, GNN) 추가는 프레임워크 범위를 통합된 순수 Rust AI 스택으로 더욱 넓힌다. 일관된 것은 방법론이다. 각 기여는 오라클과 일치한 후에만 수용되었고 재현성은 기대되는 것이 아니라 설계되고 측정되어야 할 특성으로(종종 비트 단위로) 다루어졌으며 가장 유용한 발견은 측정값이 예상에 반해 강제한 것이었다. 다음 단계는 직접적으로 이어진다. 밀집 계층을 위해 검증된 cuBLAS 백엔드를 재사용한 GPU 가속 포워드 패스, 어텐션 기반 모델을 위한 3차원 추론 패스, 그리고 런타임 감사 가능성을 그 가중치에서 빌드로 확장하기 위한 서플라이 체인 피닝이다.

## 14. 결정론적 이벤트 검출 및 분류

### 14.1 동기
크리티컬 시스템(예: 신경 의수나 산업 제어)의 실시간 이벤트 검출에는 높은 정확도뿐만 아니라 감사 가능성과 인증을 위한 절대적 결정론이 필요하다. 현재 프레임워크는 비결정론적 병렬 리덕션이나 확률적 샘플링에 의존하는 경우가 많아 하이 리스크 환경에는 적합하지 않다.

### 14.2 방법론
결정론적 슬라이딩 윈도우에 기반한 스트리밍 아키텍처를 도입한다. 사이즈 $N$의 각 윈도우 $W$는 텐서 $T \in \mathbb{R}^{1 \times N}$으로 변환된다. 이벤트 검출은 스코어 함수 $S(T) \to [0, 1]$로 정식화된다.
분류에는 SRT1 형식으로 동결된 프레임워크 코어 MLP 및 CNN 계층을 이용한다.
$$ \text{Event}(t) = \mathbb{I}(S(W_t) > \tau) $$
여기서 $\tau$는 캘리브레이션된 임계값이다.

### 14.3 결과 및 메트릭
Numenta Anomaly Benchmark (NAB)에서의 기대 퍼포먼스는 여러 스레드에 걸쳐 비트 드리프트가 제로인 상태에서 F1 스코어 $>0.85$를 목표로 한다. QSR1 int8 양자화 사용으로 에지 ARM 프로세서 상에서 지연 시간이 3배 절감되는 한편 f32 오라클 대비 MSE 비트 근접도 $<10^{-4}$가 유지될 것으로 기대된다.

## N-D 자동미분과 연구 기반 확장

2차원 역방향 모드 테이프를 넘어, SciRust는 이제 모든 연산자가 유한차분 그래디언트
체크로 검증되는 **N-D 자동미분 테이프**와 그 위에 연구로 뒷받침된 딥러닝 스택을
제공합니다. 각 기능은 특정 논문에 대응하며 테스트와 함께 제공됩니다. 전체 대응
관계(20개 중 14개 완료)는 `docs/RESEARCH_ROADMAP.md`에서 추적합니다.

- **인과적 디코더 언어 모델**, 엔드투엔드 학습(토큰 및 학습 위치 임베딩, 인과적
  멀티헤드 어텐션, 융합되고 수치적으로 안정적인 softmax 교차 엔트로피). 시퀀스를
  정확히 과적합합니다.
- **LLaMA 계열 레이어**: RMSNorm, SwiGLU, Pre-RMSNorm LLaMA 블록, RoPE(상대 위치
  성질 검증), 그룹/멀티쿼리 어텐션.
- **결정론적 옵티마이저**: Adam, AdamW, Lion, Muon(Newton–Schulz), Schedule-Free, AdEMAMix 및 SOAP(Shampoo의 고유기저에서의 Adam).
- **인증 가능한 AI**: 구간 경계 전파(IBP) **및 CROWN**(선형 완화 기반의 더 조밀한 경계)이 증명 가능한 출력 경계와 견고성 인증서를 제공.
- **재현 가능한 리덕션**: 순서 무관 합/평균/내적, 스레드 수와 무관하게 비트 단위 동일.
- **추론**: 정확한(출력 보존) 추측 디코딩, 타일형 온라인 softmax FlashAttention, DeltaNet 델타 규칙 선형 어텐션 레이어, Mamba 선택적 상태공간 레이어, RetNet 리텐션 레이어, GLA 게이트 선형 어텐션 레이어, 그리고 HGRN 게이트 선형 RNN 레이어.
- **과학적 가교**: RK4 솔버를 통해 역전파하는 Neural ODE, 그리고 PDE 잔차를 손실에 넣어 경계값 문제를 푸는 물리 정보 신경망(PINN).
- **압축**: Wanda 가지치기(활성화 인식)와 SmoothQuant, 그리고 GPTQ(2차 오차 피드백 기반 int8 가중치 양자화, CLI `scirust gptq`), 그리고 AWQ(활성화 인식 기반 탐색 방식 int8 가중치 양자화, CLI `scirust awq`).

두 가지 CLI 명령이 이를 노출합니다: `scirust certify`(IBP **및 CROWN** 경계를 나란히, 견고성)와
`scirust lm --opt adam|adamw|lion|schedule-free|ademamix|soap|lookahead|lamb|adan`(N-D 디코더 LM 학습).

세 번째 명령 `scirust conformal`은 분포 가정 없이 커버리지를 보장하는 컨포멀 예측 구간을 생성합니다.
