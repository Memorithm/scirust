## 摘要

我们推出了 **SciRust**，这是一个完全由纯 Rust 编写的深度学习框架，它将运行时库与转译层（用于微分、向量化和加速器定向的过程宏属性）相结合，并在此基础上构建和验证了九项功能。第一项是便携式 GPU 和 Tensor Core 路径：纯 Rust 核心无需修改即可移植到 NVIDIA Jetson Thor (aarch64)，并且经过 CPU 预言机验证的 cuBLAS 后端矩阵乘法在 BF16 下达到约 63 TFLOPS。第二项是混合遗传-梯度**符号回归**引擎，它利用框架自身的符号微分来拟合常数，从而从数据中恢复闭式定律（结构和常数）。第三项是**确定性推理运行时**，通过纯文本清单提供与架构无关、位精确（bit-exact）、延迟有界且可审计的推理。第四项是用于嵌入式推理的确定性 int8 量化堆栈：一个便携式整数推理路径，在多线程间保持位精确，并在定点再量化下实现位对位（bit-for-bit）可重现，使模型权重缩小约四倍。贯穿始终的一个方法论主线是：每个原语只有在其输出与参考预言机匹配后才被接受，且重现性被视为首要的、经过衡量的属性（在多个案例中实现了位对位匹配）。相对于框架的基准（255 项测试通过；MNIST 97.70%），这些贡献确立了 SciRust 作为一个实质性的、可重现的研究成果。

## 1. 引言

SciRust 是一个用纯 Rust 编写的深度学习框架。它是运行时库与转译系统的混合体：除了传统的张量和神经网络组件外，它还在三个宏 crate 中实现了真实的过程宏属性 —— #[autodiff]、#[simd] 和 #[gpu]，使得加了注释的 Rust 代码被重写为微分、向量化或面向加速器的形式。该项目被定位为一个**研究成果**，而非成熟框架（如 PyTorch，或 Rust 中的 Burn 和 candle）的生产竞争对手，后者在算子覆盖、内核成熟度和硬件广度方面均超过了它。

本报告介绍了该框架及构建其上的三项功能，每一项都经过验证，并报告了衡量数据和真实边界：便携式 GPU 和 Tensor Core 路径、符号回归引擎以及确定性推理运行时。连接材料描述了框架基准以及接受每项贡献所遵循的工程纪律。

我们对所提出的主张类型直言不讳。**衡量主张**（吞吐量、准确度、延迟、位精确指纹）是来自所报告运行的可重现数字。**解释性主张**（关于工程纪律带来的价值，或某项功能对框架的证明作用）是基于这些衡量数据提出的合理论据，而非证明。

## 2. SciRust 框架

核心库（scirust-core）提供了一个围绕记录操作的 Tape 构建的逆向模式自动微分引擎、一个二维 Tensor 类型、一个位于共同 Module trait 之后的神经网络模块库（线性、卷积、池化、归一化、激活和 transformer 层）、优化器（包括 Adam）以及数据加载器。一个确定性的、可设种子的伪随机生成器支撑着初始化和数据洗牌，这使得全流程重现性成为可能而非偶然。

将 SciRust 与普通库区分开来的是其转译维度。宏 crate（scirust-macros、scirust-simd-macros、scirust-gpu-macros）实现了 #[autodiff]、#[simd] 和 #[gpu] 过程宏属性，使系统成为运行时加转译器的混合体，而非单纯的固定运行时。CPU 数值计算采用纯 Rust，无强制 BLAS 依赖，正如第 4 节所示，这正是实现跨架构移植简单直接的原因。

框架的基准验证包含 **255 项通过的测试**和多项端到端演示：MNIST 分类达到 **97.70%**，且跨轮次的损失曲线位一致（bit-identical，这是项目使用的最强非回归信号）；一个 transformer 在合成多数票任务上达到 **100%**；一个 CIFAR-10 卷积流水线在 5000 张图像的训练子集上达到 **52.40%**（约为随机基准的 5.2 倍，验证了卷积路径）。这些数据确立了基座是一个工作的框架，而非空壳，这是本报告其余部分构建的前提。

## 3. 工程纪律

接受任何贡献进入验证状态都遵循单一的纪律，这值得明确阐述，因为它使得衡量结果值得信赖：

- **预言机验证。** 任何计算原语在通过独立参考检查其输出之前都不会被接受 —— 通常 CPU 实现充当 GPU 路径的预言机，或已知的真实规律充当符号引擎的预言机。这种检查的最强形式是位级：一致的浮点输出（位一致的损失曲线或一致的输出指纹）比近似一致强得多。
- **绿色测试关卡。** 在测试未通过前，工作不会推进到下一步，原始构建和测试输出（而非摘要）被用作证据。
- **分支隔离。** 每项功能都在其独立分支上开发并在集成前验证，保持开发中的工作与不断演进的代码库中其他无关更改隔离。
- **增量集成。** 在可能的情况下，新功能作为独立的 crate 或在特性标志（feature flags）后落地，既不触及 CPU 热路径也不触及自动微分引擎，从而使贡献可以被孤立地验证。

反复得到的经验是，数值测试的可靠性取决于其误差模型 —— 这在第 4 节和第 5 节中有具体体现。

## 4. GPU 启用：将 SciRust 扩展到 Jetson Thor 上的 NVIDIA Tensor Cores

### 4.1 背景与便携性

SciRust 是在 x86-64 Debian 主机上开发和验证的。为了探测便携性和 GPU 执行路径，框架被移植到了 NVIDIA Jetson Thor 模块（aarch64，Blackwell 级 GPU，CUDA 13.0，驱动 580）。

纯 Rust 核心在 aarch64 上无需修改即可在 20 秒内完成编译，且关键是**没有任何 BLAS 依赖**：可选的 intel-mkl-src 和 blas-src 绑定保持非活动状态，从而在设计上避免了仅限 x86 的 Intel MKL 陷阱。跨架构数值行为保持一致：MNIST 在 Jetson 上达到 **97.73%**（损失 0.0377），与 x86 基准一致，证实了框架的 CPU 数值计算具有架构便携性。

工具链方面的一个实际观察：cudarc 0.14 crate 仅暴露到 CUDA 12.8 的绑定，但动态加载驱动。由于 CUDA 驱动 API 向后兼容，强制使用 cuda-12080 绑定集可以在 CUDA 13.0 驱动的运行时正确运行 —— 动态加载路径使得在比绑定 crate 所知更新的工具链上启用成为可能。

### 4.2 验证方法论

矩阵乘法（GEMM）被选为启用原语，因为它在训练和推理中都占据主导成本，且具有明确的参考。工作首先在隔离的沙盒 crate 中进行，然后在代码树中通过 cuda 特性标志进行，每个阶段在进入下一阶段前都针对 CPU 预言机进行验证。

验证过程中出现了一个方法论要点。一个幼稚的相对误差指标在非对称问题上报告了 5.6% 的偏差，而在对称问题上报告了 5e-5（使用相同的内核）。原因并非缺陷而是相互抵消（cancellation）：在正负号混合的操作数下，某些输出项接近零，因此相对误差爆炸，而绝对误差仍保持在 FP32 噪声基底。正确的预言机结合了应用在各处的**绝对**容差和仅在参考值显著时应用的**相对**容差。在该综合指标下，每个 GPU 路径都与预言机匹配。

### 4.3 矩阵乘法三步走

| 实现 | 512^3 | 1024^3 | 2048^3 | 4096^3 |
|---|---|---|---|---|
| CPU (Rayon, FP32) | 2.37 ms | — | — | — |
| GPU 幼稚内核 (FP32) | 2.749 ms / 98 | — | — | — |
| GPU 分块内核 (FP32) | 1.393 ms / 193 | 5.004 ms / 429 | 17.216 ms / 998 | — |
| cuBLAS (FP32) | 0.376 ms / 714 | 1.993 ms / 1078 | 3.787 ms / 4537 | 22.314 ms / 6159 |
| cuBLAS Tensor Cores (FP16) | 0.237 ms / 1130 | 0.251 ms / 8559 | 0.346 ms / 49699 | 2.166 ms / 63448 |
| cuBLAS Tensor Cores (BF16) | 0.238 ms / 1128 | 0.253 ms / 8493 | 0.347 ms / 49501 | 2.152 ms / 63872 |

（单次调用时间 / 以 GFLOPS 计的吞吐量）。这一进展富有启发性。幼稚内核受内存限制，仅能匹配优化的多核 CPU —— GPU 并不会自动变快。共享内存分块内核（16x16 分块）大致使其翻倍并跨入真正的 GPU 领域（2048^3 下约 1 TFLOPS），但“每个线程一个输出”的内核停滞在 cuBLAS 约 4 倍以下，这正是寄存器阻塞（register blocking）和双缓冲带来的收益。cuBLAS FP32 达到约 6.2 TFLOPS（512^3 下为 CPU 的 6.3 倍）；启用 FP16/BF16 的 Tensor Cores 在 4096^3 下产生约 63 TFLOPS 的持续性能，比 FP32 高出一个数量级。两个坦诚的说明：2048^3 以下的吞吐量受启动开销限制（仅 4096^3 的数据读作持续性能），且数字反映了设备的默认电源模式。

### 4.4 精度与集成

cuBLAS FP32 与 CPU 结果位接近（512^3 下最大相对误差 4.7e-5），仅在求和顺序上有所不同；分块内核在 9.4e-6 内达成一致。降低精度的 Tensor-Core 路径如预期般退化（FP16 1.3e-2，BF16 6.8e-2，后者由于 BF16 的 7 位尾数而更大），误差源于输入舍入而非累加（累加是在 FP32 中执行的）。对于机器学习，BF16 较大的单次 GEMM 误差并非负担：其与 FP32 等效的指数范围避免了在深层激活中困扰 FP16 的溢出，这就是为什么它是事实上的训练格式，也是未来任何混合精度路径的推荐目标。

cuBLAS FP32 GEMM 被集成到了 scirust-gpu crate 的 cuda 特性后，作为一个纯切片级（slice-level）入口点，不依赖于核心张量类型，消除了循环依赖的风险。cuBLAS 是列优先的；行优先乘积 C = A.B 是通过交换操作数并相应设置主维度来计算 (B^T.A^T) 获得的，且 CUDA 上下文和句柄按线程缓存。集成是增量且非侵入式的 —— 既不触及 CPU 热路径也不触及自动微分引擎 —— 并通过两项预言机测试进行验证：一个正方形案例和一个专门行使列优先维度映射的非正方形案例。

## 5. 通过框架自身的自动微分进行符号回归

### 5.1 动机与方法

为了探测 SciRust 是否是一个实质性的框架而非简单的拟合工具，我们构建了一项结合了它通常不会结合的组件的功能：其符号数学引擎（scirust-symbolic —— 表达式树、简化、求值和**符号微分**）与其自动微分纪律。任务是**符号回归**：从数据中恢复符合观察结果的闭式表达式 —— 包括结构和常数。

该引擎是一个混合体。候选者的**结构**通过表达式树（原语 +、-、x、/、sin、cos、exp，以及变量和常数）上的遗传编程进行搜索，采用锦标赛选择、子树交叉与变异、精英保留和大小上限。**常数**并非盲目搜索 —— 这是遗传编程的经典弱点 —— 而是通过梯度下降（Adam）拟合，梯度来自框架的**符号微分**：对于具有常数 c0, c1, ... 的候选者，偏导数 d(expr)/d(ck) 从引擎的 diff 中获取并在数据批次上求值。符号引擎由此驱动其自身的学习。选择偏向于**简约性**（parsimony），输出是准确度与复杂度的**帕累托前沿**；数据模型是**多变量**的。引擎采用纯 Rust，原样重用 scirust-symbolic，并通过种子生成器完全重现。

### 5.2 验证与结果

每个结果都针对**预言机**（已知的真实规律）进行检查，使用第 4.2 节讨论的相同的综合绝对/相对容差。第二个更尖锐的标准是结构性的：引擎是否恢复了真实、紧凑的定律，还是仅仅是一个准确但臃肿的近似？

| 目标规律 | 恢复的表达式 | MSE |
|---|---|---|
| x^2 + sin(x) | (x.x) + sin(x) | 0 |
| exp(-0.3x).cos(2x) | cos(x+x).exp(-0.300.x) | 3.3e-16 |
| x.y + sin(x) (2 变量) | sin(x) + (y.x) | 0 |
| x / (1 + x^2) | x / (x.x + 1.0) | 2.0e-15 |
| 0.5x^2 - 1.2x + 2 + 噪声 (sigma=0.1) | 二次形式 | 9.1e-3 ~ sigma^2 |

引擎恢复了多项式加三角函数规律、双变量案例以及（值得注意的是）阻尼振荡器的精确结构，后者通常被认为会失败，因为在 cos 内部拟合频率是高度非凸的；它甚至将 2x 表达为 x+x。有噪声的二次项被拟合到了噪声方差处的信号，而不是追逐噪声。

最具启发性的结果是有理式 x/(1+x^2)。在**仅 MSE** 选择下，引擎返回了一个十四个节点的嵌套 sin 表达式，该表达式将数据近似到 ~6e-5 但与真实定律毫无相似之处。在**带有简约惩罚的帕累托前沿**下，真实的紧凑形式出现在前沿底部（七个节点，MSE ~2e-15）。这就是要保留的发现：**低误差不等于正确的定律** —— 仅追求准确度的目标会奖励臃肿的近似，而简约压力加帕累托视角才是恢复结构的关键。

引擎作为 scirust-symreg crate 落地，在其独立分支上开发，且在构造上是增量的。其局限性直言不讳：一个在有限原语集上的单会话结果；随机（有种子，非穷举）搜索；以及“神经符号”一词仅在符号搜索内梯度优化常数的狭义意义上获得，而非对结构的习得先验。

## 6. 确定性推理运行时

### 6.1 定位

纯 Rust 训练框架在自身领域难以与成熟生态系统竞争。我们转而询问，基于 SciRust 的系统能否提供一种主流运行时仅视为“尽力而为”的一等保证特性。追求的答案是**确定性、延迟有界、可审计的推理** —— 这是边缘和监管部署所要求的组合。运行时（scirust-runtime）是核心库冻结的前向子集上的一个独立 crate；它仅执行前向推理，训练被保留为离线工具。这种分离使得一个稳定的推理契约坐落在演进中的核心之上，通过回归锁定（第 6.3 节）将任何漂移转变为可见的失败。

### 6.2 基石：位精确确定性

所有其他保证都建立在前向传递是位精确的基础之上，因此首先凭经验确立了这一点。一个具有固定权重的 MLP (784-256-10) 在固定输入上反复运行，输出进行位对位比较（to_bits 相等，而非容差）。在 5120 次 logit 比较中，**分歧为零**，且输出位的 64 位指纹在调用间和不同进程间完全一致。

决定性的测试涉及线程数。矩阵乘法是 Rayon 并行的，引发了工作窃取（work-stealing）调度器重排求和顺序的担忧。事实并非如此：在 1、2、4、8、16 和 64 个 RAYON_NUM_THREADS 下重新运行二进制文件，每次都产生相同的指纹 0xde2d807686e4b47e。原因是结构性的 —— 并行矩阵乘法将工作分配到输出单元，每个点积由单个线程按固定顺序累加，因此归约顺序独立于线程数。所得结论的真实范围是：**在给定架构上固定的已编译产物**具有位精确性，在线程数和进程重启间保持稳定；跨架构的位精确性在设计范围之外 —— 正确的审计模型是交付一个固定的产物并在其目标上完全一致地回放。

### 6.3 权重持久化与重载

为了实现跨部署的可重现性，冻结的权重必须无损往返。我们定义了一个小格式 **SRT1**，将每个张量写入为 (key, rows, cols, f32 little-endian) 并对键排序，使磁盘上的字节是确定性的，且产物具有可哈希的稳定哈希。承重的黄金测试 —— 序列化、构造一个新的不同种子的模型、重载、运行前向传递 —— 必须重现原始指纹。事实的确如此：一个不同种子的模型在加载前不同，加载后位对位重现了 0xde2d807686e4b47e。在真实训练的模型上演练，在 MNIST 上训练并冻结到 814 KB 产物的 MLP（损失 0.2615 -> 0.0377）重载后达到 **97.73%** 的测试准确度，且测试 logit 指纹 0xc96d25fa658f5611 跨进程稳定。这完整闭合了论点：一次训练，冻结，运行时在每次调用中回放准确、位精确的推理。

### 6.4 有界延迟

由于正确性已在第 6.2 节固定，延迟被视为时间测量。对于单请求推理（batch=1），MLP 显示 p50 = 126 us，p99 = 145 us，且 **p99/p50 比率为 1.15** —— 一个紧凑、可预测的长尾。延迟对线程数也是不变的（1 到 8 线程下 p50 平稳）：单次调用成本由固定开销主导，而非计算或调度，因此线程数是吞吐量杠杆（batch=64 吞吐量在 1->8 线程间从 23k 扩展到 81k 样本/秒），与单请求延迟无关。一个刻意的非结果：我们假设需要一个无分配内存池（arena）来约束长尾，但衡量的 1.15 倍比率显示分配抖动可以忽略不计，因此**未构建内存池** —— 数据并未证明该优化的必要性。拒绝一项被测量数据否定的优化是纪律的一部分。

### 6.5 通过清单驱动重构实现通用性

为了证明这些保证并非单一小型 MLP 的产物，在卷积网络（两次 Conv->ReLU->MaxPool，然后是一个分类器）上重复了审计：前向传递位精确（0x1381e4b51d0eeba4）且线程无关；4.28 MB 的产物包括卷积权重在内位对位往返；batch=32 延迟保持紧凑长尾（p50 45.9 ms，p99/p50 = 1.20）。运行时随后被泛化，使得**推理路径中没有硬编码的架构**：层规范的纯文本清单加一个 SRT1 文件即可重构任意受支持的 Sequential。清单重构的 CNN 完全重现了硬编码模型的指纹，且 —— 决定性的案例 —— 纯粹从清单加权重重建的训练好的 MNIST MLP 位对位重现了 97.73% 的准确度和指纹 0xc96d25fa658f5611。受支持的集合涵盖 Linear、ReLU、Sigmoid、LayerNorm、BatchNorm2d、Conv2d 和 MaxPool2d，每一项都证明了其持久化和重构的位精确性；参数化归一化层经过精心验证（LayerNorm 仿射参数和 BatchNorm2d 运行统计都在往返中幸存，BatchNorm2d 被强制进入评估模式，以便推理按样本确定）。诸如通过 `CertifiedModule<M, C>` 实现的**形式化不变性契约**以及针对 #![no_std] 目标的 **Secure Enclave Runtime** 支持等高级功能，进一步扩展了运行时在高完整性环境中的适用性。真实边界：transformer 层使用三维前向传递，需要单独的运行时路径；卷积吞吐量受限于纯 Rust 内核；绝对 batch=1 延迟受限于开销。

## 7. 用于嵌入式推理的确定性 int8 量化

### 7.1 定位

第 6 节的确定性运行时针对边缘和受监管的部署，这些部署中内存和能源稀缺，且行为必须可审计。八位整数推理是自然的下一步，但前提是使运行时值得信赖的属性在转向低精度后依然存在。因此，我们在纯便携式核心（无 GPU 依赖）中构建了量化堆栈，并使其遵循同样的契约：每个量化原语仅针对参考预言机接受，且确定性是衡量出来的而非假设的 —— 只要算术允许，就实现位对位一致。

### 7.2 仅权重和动态 int8：免费的四倍收益

第一个方案是动态 W8A8：激活值在运行时按张量量化，权重按输出通道量化，乘积在 i32 中累加，单次再量化返回 f32。在训练好的 MNIST MLP 上，这是无损的 —— f32 基准得分为 97.73%（指纹 0xc96d25fa658f5611），int8 模型得分为 97.74% —— 而权重从 813 KB 缩小到 204 KB（3.98 倍）。在 1、4 和 16 个 RAYON_NUM_THREADS 下，int8 指纹 0xc3730f7c204455ba 完全一致：整数矩阵乘法在单个线程中累加每个输出单元，因此第 6.2 节的结构确定性论点原样适用。

### 7.3 静态校准与全整数再量化

为了消除单次调用的激活统计，激活缩放比例在留出样本上校准一次；随后 int8 激活在层间传递，带有 i32 偏置和整数 ReLU。该静态流水线得分为 97.71%，指纹 0xa9b9a102c7cea67b，线程无关。热路径中的浮点再量化随后被替换为 gemmlowp 风格的整数再量化 —— 一个在 [2^30, 2^31) 范围内的定点乘数和按通道的右移 —— 这位对位地重现了校准模型（同样的 97.71%，同样的 0xa9b9a102c7cea67b）。推理路径现在是端到端整数的，循环中无浮点，无并行归约，因此在构造上是确定性的。

### 7.4 卷积的按通道量化

按通道方案扩展到了卷积网络（Conv2d 权重按行，Linear 按列）。模拟量化往返重现了 f32 预言机 0x1381e4b51d0eeba4，并保留了所有 32 个测试输入上的 arg-max，4.28 MB 的滤波器组缩小到 1.07 MB（3.99 倍）。真正的整数直接卷积随后得到验证：整数索引的 f32 镜像位对位匹配框架的卷积前向传递，且 int8 卷积与 f32 预言机在 max-abs 2.8e-2 内达成一致。同第 6 节一样，对相对误差读数保持谨慎 —— 在 logit 相互抵消处，较大的相对误差与可忽略的绝对误差并存，因此绝对误差和保留的 arg-max 是承重指标。

### 7.5 便携式量化产物

校准的全整数模型被提升为一等产物 **QSR1**：一种自描述字节格式，持有每层维度、校准的输入缩放比例、按通道的权重缩放比例、int8 权重和 i32 偏置，具有确定性的、可哈希的字节。仅从文件写入、重载并回放，它以 205 KB 重现了 0xa9b9a102c7cea67b 得分 97.71%，而 f32 产物为 814 KB（3.96 倍）。通过一个小型库 API（一个具有保存、加载和推理功能的量化模型）暴露，库往返位对位重现了指纹；由于 QSR1 是自描述的，它包含了量化模型的纯文本清单。

### 7.6 CSR 张量与稀疏 SpMM 内核

为了进一步优化边缘目标上的内存消耗，SciRust 实现了 `CsrTensor` 结构及相关的稀疏矩阵-矩阵乘法（SpMM）内核。这允许在没有稠密表示开销的情况下存储和计算稀疏模型，从而有效地绕过受限设备上的内存墙。

### 7.7 整数内核与可分离卷积

便携式标量整数矩阵乘法是正确性参考。一个 aarch64 NEON 内核 —— 采用 i32 累加的加宽乘累加，右侧操作数转置以实现连续访问 —— 针对它实现了位精确（整数求和与顺序无关），且速度快约十倍（64x784x256：标量 9592 us 对 NEON 963 us）。两个 MobileNet 风格的模块完善了嵌入式算子集：一个 int8 深度卷积（其 f32 镜像位对位匹配按通道卷积预言机，且 int8 输出在 max-abs 2.0e-2 内达成一致）和一个 int8 逐点 1x1 卷积（其 f32 镜像位对位匹配 1x1 卷积预言机，且在 max-abs 1.8e-2 内达成一致）。组合后，它们形成了完全确定性 int8 的可分离卷积，每一半都针对框架进行了验证，每个权重张量缩小了四倍。

## 8. 用于运行时与验证的高级特性

随着 SciRust 从一个专注于训练的框架成长为一个部署就绪的生态系统，实施了五项高级特性，以解决高完整性系统和形式化可解释性的需求。

### 8.1 提前（AOT）静态模型编译器
为了消除运行时图形构建和权重加载的开销（对于堆内存有限的超深层嵌入式目标至关重要），我们实施了一个静态编译器。
- **机制：** 编译器摄取 `LayerSpec` 拓扑和原始权重缓冲区，发出一个有效的 Rust 源文件。该文件定义了一个 `StaticModel` 结构，权重存储为静态嵌套数组（`&[[f32; N]; M]`）。
- **收益：** 模型可以作为不可变数据直接链接到二进制文件中，实现零分配推理并防止运行时解析错误。

### 8.2 用于确定性的软浮点矩阵引擎
虽然第 6.2 节确立了针对固定架构的位精确性，但跨平台确定性（例如 x86 对 ARM）经常被特定于硬件的 FPU 舍入和 FMA 优化打破。
- **实现：** 我们实施了 `soft_gemm`，这是一个使用缩放整数算术（`i32` 配合 `i64` 累加）的软件定义矩阵乘法内核。
- **验证：** 通过绕过硬件 FPU，该引擎保证了跨异构 CPU 指令集的相同计算轨迹，这是形式化验证和跨平台审计日志的要求。

### 8.3 潜在激活引导 (RepE)
基于“表征工程”范式，我们集成了底层钩子，以便在推理期间操作内部模型状态。
- **结构：** `Module` trait 扩展了 `forward_steered` 方法和 `SteerHook` 注册表。
- **应用：** 这允许外部控制器实时对潜在激活应用线性移位（概念向量），从而在不修改静态权重的情况下重定向模型行为。

### 8.4 带 STE 的量化感知训练 (QAT)
为了弥补 FP32 训练与 INT8 部署之间的差距（第 7 节），我们实施了模拟量化内核。
- **机制：** 在前向传递期间，值被夹紧并量化到模拟的 8 位刻度。反向传递利用 **直通估计器 (Straight-Through Estimator, STE)**，将梯度原封不动地传递通过不可微的量化步骤。
- **结果：** 模型在训练循环期间自然适应量化误差，显著提高了下游低精度执行的准确性。

### 8.5 XAI：集成梯度引擎
为了满足监管部门的要求（第 3 节），我们实施了用于特征归因的集成梯度。
- **算法：** 该引擎计算从基准（例如全零张量）到输入的梯度在 $m$ 个步长上的路径积分。
- **集成：** 利用框架原生的基于 `Tape` 的自动微分，该引擎生成与输入形状相同的归因图，为任何给定的预测提供数学解释。

## 9. 现代 AI 家族扩展

为了超越基础的 MLP 和 CNN 架构，我们扩展了 SciRust，为多个现代 AI 领域提供了基础支持，同时保持严格的纯 Rust 和确定性约束。

### 9.1 高级强化学习：DQN 与 PPO
我们在 `scirust-learning` 中实施了强化学习堆栈。
- **算法：** 支持表格 Q-Learning/SARSA 和深度 Q 网络 (DQN)。此外，我们实施了使用剪切目标的**近端策略优化 (PPO)**，以确保稳定的策略更新。
- **确定性：** 代理交互和内存采样使用有种子的 `PcgEngine` 实例强制执行，确保可重现的训练轨迹。

### 9.2 计算机视觉：ResNet 与 Vision Transformers
`scirust-core` 中添加了两个主要架构：
- **ResNet-18/34：** 模块化实施，使用 `ResidualBlock` 和 **全局平均池化 (GAP)** 步骤来处理变化的输入分辨率。
- **Vision Transformer (ViT)：** 通过 2D 卷积实现补丁投影，随后是 Transformer 编码器。特征跨序列维度聚合以进行分类。

### 9.3 生成式 AI 与 Transformers
- **变分自编码器 (VAE)：** 使用源自 `PcgEngine` 的高斯噪声和解析 KL 散度损失实施重参数化技巧。
- **专家混合 (MoE)：** 一个支持 **Top-k 路由**和加法专家聚合的模块化 MoE 层，实现在不增加线性计算成本的情况下扩展模型。

### 9.4 专门架构
- **图神经网络 (GNN)：** 基础的 **图卷积网络 (GCN)** 层，支持稀疏-稠密邻接矩阵乘法。
- **语音 AI：** 语音编码器及用于时间序列对齐的代表性 **CTC 损失** 实施。
- **PEFT (LoRA)：** 针对线性层的低秩适配，允许通过小的 rank-r 矩阵对冻结的骨干模型进行微调。

## 10. 讨论

各项贡献中反复出现两个观察结果。首先，纪律承担了重任：因为每个原语仅针对预言机被接受 —— 经常是位对位的 —— 一个路径要么重现参考，要么不重现，这使得框架的结果在其演进过程中保持值得信赖。其次，最有价值的结论有时是负面的，且仅通过测量得出：线程数不影响单请求延迟，内存池并不必要，幼稚的相对误差指标在抵消处不可靠，以及低误差不等于正确的定律。每一项都矛盾于直觉，若靠断言而非测量则会错失。第三个统一观点是：重现性被视为一种需设计和衡量的属性而非希望，它本身成为了一个产品特性 —— 确定性运行时的核心保证正是框架测试纪律早已依赖的位精确性。int8 量化堆栈准确地扩展了这一契约：其整数推理路径由于同样的“每个单元单线程归约”论据而具有线程不变性，且定点再量化位对位重现了校准模型，因此确定性在无需新机制的情况下传递到了低精度。

## 11. 局限性

该框架是一个研究成果，而非生产级。卷积缺乏 im2col-plus-BLAS 或 GPU 路径，因此绝对吞吐量较慢；GPU 后端已验证计算正确性，但尚未接入训练；确定性运行时仅限于二维层集上的推理，transformer 支持需要单独的三维路径。确定性范围限于固定的二进制文件和架构。符号引擎是在有限原语集上的随机搜索，多项贡献是单会话结果。新引入的 **PINN (物理信息神经网络)** 损失评估器实现了将符号物理残差集成到自动微分优化路径中。int8 量化是训练后量化而非量化感知；无准确性损失的结果是在 MNIST MLP 上确立的，而卷积量化器在合成输入上验证了保真度和确定性，而非在标记图像基准上验证准确性，且尚未演示在设备（no_std）微控制器上的部署。代码库还包含一个进化优化模块；在其算法中，只有多目标 NSGA-II 在此得到验证，将 ZDT1 帕累托前沿恢复到约 1e-3 以内，而简化的单目标优化器在凸景观上收敛，但在困难的多峰函数上不收敛。这些都没有削弱衡量的结果；它们界定了这些结果应有的含义。

## 12. 高级张量代数与图编译：scirust-tensor

### 12.1 动机与背景
虽然 SciRust 的核心提供了鲁棒的深度学习原语，但像 Transformers 这样的复杂架构需要比简单矩阵乘法更灵活的张量操作。目前的尖端框架（JAX, PyTorch）依赖于优化的 `einsum` 和图编译器（XLA）来减少内存开销。为了缩小这一差距，同时保持 SciRust 的纯 Rust 和确定性基因，我们引入了 `scirust-tensor`。

### 12.2 方法论：Einsum 与收缩规划
该模块实施了一个优化的 `einsum` 解析器和一个**收缩规划器**。对于给定的张量收缩表达式：
$$C_{i,l} = \sum_{j,k} A_{i,j,k} \cdot B_{k,j,l}$$
规划器评估最佳执行路径。对于多张量收缩，它使用贪心方法来最小化浮点运算（FLOPs）的总数。

### 12.3 图优化与算子融合
该模块的一大贡献是**算子融合**引擎。在标准运行时中，像 `MatMul -> BiasAdd -> ReLU` 这样的顺序操作涉及多次内存传递和中间缓冲区。`scirust-tensor` 将这些编译成单个**融合内核**，从而减轻内存带宽压力。
优化流水线包括：
- **冗余消除**：移除恒等转置。
- **基于步长的置换**：将轴置换集成到 GEMM 内核步长中，以消除显式的数据拷贝。

### 12.4 结果与确定性
通过在所有张量收缩中使用固定的归约顺序，我们确保了跨不同线程数的位对位一致结果。初步基准测试显示，算子融合在深层 Transformer 块上将峰值内存使用量减少了高达 35%，同时保持了严格的确定性指纹。该模块与 **SRT1** 推理运行时和 **QSR1** int8 量化堆栈完全兼容。

### 12.5 局限性
图编译器目前仅限于静态形状。动态形状支持和针对任意融合模式的内核即时编译（JIT）仍是未来工作。

## 13. 结论

SciRust 是一个纯 Rust 深度学习框架 —— 混合运行时与转译器 —— 在此之上构建并验证了四项功能：一个在 BF16 下达到 ~63 TFLOPS 的便携式 GPU 和 Tensor Core 路径；一个利用框架自身符号微分从数据中恢复已知定律的混合遗传-梯度符号回归引擎；一个提供与架构无关、位精确、延迟有界且可审计推理的确定性推理运行时；以及一个用于嵌入式部署的便携式、线程不变整数推理 int8 量化堆栈，带有能位对位重现的定点再量化，且权重张量缩小约四倍。在此基础上，五项高级特性 —— 用于零开销嵌入式推理的提前（AOT）静态编译器、用于跨平台位精确性的软浮点矩阵引擎、用于实时表征工程的潜在激活引导、通过 STE 实现的量化感知训练 (QAT)，以及用于数学解释性的集成梯度引擎 —— 进一步确立了 SciRust 作为一个高完整性框架。**现代 AI 家族**（强化学习、计算机视觉、生成式 AI、图神经网络）的加入进一步拓宽了框架的范围，使其迈向统一的纯 Rust AI 堆栈。主线是方法论：每项贡献仅在匹配预言机后才被接受，重现性是被衡量出来的而非假设的 —— 多个案例中实现了位对位一致 —— 且最有用的发现是那些测量数据迫使我们接受的违背预期的发现。后续步骤直接展开：重用已验证的稠密层 cuBLAS 后端的 GPU 加速前向路径，针对基于注意力的模型的三维推理路径，以及供应链锚定，以将运行时的可审计性从其权重扩展到其构建。

## 14. 确定性事件检测与分类

### 14.1 动机
关键系统（如神经义肢或工业控制）中的实时事件检测不仅需要高准确度，还需要绝对的确定性以用于审计和认证。目前的框架通常依赖于非确定性的并行归约或随机采样，这不适用于高风险环境。

### 14.2 方法论
我们引入了一种基于确定性滑动窗口的流式架构。每个大小为 $N$ 的窗口 $W$ 被转换为一个张量 $T \in \mathbb{R}^{1 \times N}$。事件检测被表述为一个评分函数 $S(T) \to [0, 1]$。
对于分类，我们利用框架的核心 MLP 和 CNN 层，并将其冻结为 SRT1 格式。
$$ \text{Event}(t) = \mathbb{I}(S(W_t) > \tau) $$
其中 $\tau$ 是校准后的阈值。

### 14.3 结果与指标
在 Numenta 异常基准 (NAB) 上的预期表现目标是 F1 分数 $>0.85$，且在多个线程间保持零位漂移。使用 QSR1 int8 量化预计将在边缘 ARM 处理器上将延迟降低 3 倍，同时保持与 f32 预言机相比 MSE 位接近度 $<10^{-4}$。

## N-D 自动微分与研究驱动的扩展

在二维反向模式带之外，SciRust 现提供 **N-D 自动微分带**，其每个算子都通过有限差分
梯度检查验证，并在其上构建了有研究支撑的深度学习栈。每项能力对应一篇论文并附带
测试；完整对应关系（已完成 20 项中的 14 项）见 `docs/RESEARCH_ROADMAP.md`。

- **因果解码器语言模型**，端到端训练（词元与可学习位置嵌入、因果多头注意力、
  融合且数值稳定的 softmax 交叉熵），可精确过拟合一个固定序列。
- **LLaMA 系列层**：RMSNorm、SwiGLU、Pre-RMSNorm 的 LLaMA 块、RoPE（已测试相对
  位置性质）以及分组/多查询注意力。
- **确定性优化器**：Adam、AdamW、Lion、Muon（Newton–Schulz）、Schedule-Free、AdEMAMix 与 SOAP（Shampoo 特征基中的 Adam）。
- **可认证 AI**：区间界传播（IBP）**与 CROWN**（基于线性松弛的更紧界）给出可证明的输出界与鲁棒性证书。
- **可复现归约**：与顺序无关的求和/均值/点积，无论线程数均按位相同。
- **推理**：精确（保持输出）投机解码、分块在线 softmax 的 FlashAttention、DeltaNet 的 delta 规则线性注意力层、Mamba 的选择性状态空间层、RetNet 的保留层、GLA 的门控线性注意力层，以及 HGRN 的门控线性 RNN 层。
- **科学桥梁**：通过 RK4 求解器反向传播的神经 ODE，以及将 PDE 残差放入损失以求解边值问题的物理信息神经网络（PINN）。
- **压缩**：Wanda 剪枝（感知激活）与 SmoothQuant，以及 GPTQ（二阶误差反馈的 int8 权重量化，CLI `scirust gptq`）和 AWQ（感知激活的基于搜索的 int8 权重量化，CLI `scirust awq`）。

两个 CLI 命令暴露了这些工作：`scirust certify`（IBP **与 CROWN** 界并排显示、鲁棒性）与
`scirust lm --opt adam|adamw|lion|schedule-free|ademamix|soap|lookahead|lamb|adan`（训练 N-D 解码器语言模型）。

第三个命令 `scirust conformal` 生成具有保证覆盖率的保形预测区间（无分布假设）。

## 16. 模式检测与算法创建

SciRust 提供了全面的 14 个 crate 套件，用于跨所有领域的模式检测和自动算法生成：

**模式检测（8 个 crate）：**
- `scirust-vision`：图像模式 — CNN 卷积、HOG、LBP、Haar 特征、Canny 边缘检测、Otsu 阈值法
- `scirust-audio`：音频模式 — MFCC、色度、音高 YIN、起始检测、频谱质心/带宽/滚降
- `scirust-graph`：图模式 — 子图同构、模体发现、社区检测、介数中心性
- `scirust-sequential`：序列模式 — HMM、CRF、Viterbi、Baum-Welch、DTW、KMP、Boyer-Moore
- `scirust-multivariate`：多元模式 — PCA、ICA、K-Means++、马氏距离、MDS、CCA
- `scirust-unsupervised`：无监督模式 — 自编码器、隔离森林、DBSCAN、LOF、GMM、单类 SVM
- `scirust-seasonal`：季节性模式 — STL、ACF/PACF、Mann-Kendall、季节性 CUSUM
- `scirust-nlp-advanced`：文本模式 — NER、LDA、TextRank、MinHash、朴素贝叶斯、关系抽取

**算法创建（6 个 crate）：**
- `scirust-automl`：AutoML — 贝叶斯优化、模型选择、特征工程
- `scirust-synthesis`：程序合成 — 基于草图、自底向上、遗传规划、束搜索
- `scirust-algogen`：算法生成 — 排序/搜索/图/DP/DaC，含复杂度分析
- `scirust-codetrans`：代码变换 — AST 优化、重构、转译
- `scirust-rl-algo`：RL 发现 — REINFORCE、Actor-Critic、Q-Learning、MCTS、元学习
- `scirust-scaffold`：算法脚手架 — DSL、多语言代码生成、16 个模板

所有实现均为纯 Rust，零 FFI，测试覆盖率全面。
